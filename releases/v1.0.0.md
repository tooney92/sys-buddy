# sys-buddy v1.0.0 — 2026-07-23

First tagged release. sys-buddy is a broker for **cross-human AI agent collaboration**:
two people's coding agents (e.g. a backend and a frontend) coordinate through a shared,
prompt-injection-hardened broker instead of pasting things between chats.

## Highlights
- **Broker over MCP (HTTP)**, local or remote; remote mode auths agents by scoped bearer
  token (single-use invites, `rotate_token`).
- **Onboarding**: invite links, a browser join page, and a **desktop app** (`sys-buddy gui`)
  for hosting a task / joining as a buddy / wiring Claude.
- **Pre-flight readiness** quiz gates an agent's tools until it proves it read the rules.
- **Rules of Engagement**: peer messages are DATA; the only fetchable URL is the signed
  `staging_url` — the standing defense against agent-to-agent prompt injection.
- **Messaging**: directed or broadcast, with wait/check/ack.
- **Contracts**: `propose_contract` → `lock_contract` (all roles) → `get_contract`, versioned
  and immutable once locked; `reopen_negotiations` to re-plan.
- **Lifecycle** via `report_status`, and a **read-only dashboard** with live SSE updates.

## Notable behavior in this release
- **`get_contract` shows the proposed contract, not just locked ones.** Before a contract
  locks it returns `status: "proposed"` with the interface shape and signature status; the
  **`staging_url` is withheld (null) until every role signs**. Review it there, then
  `lock_contract(version)` to sign. This is the fix for the recurring "I can't see the
  contract to review it before signing" deadlock.

## Known limitations / on the roadmap (`v2.md`)
- `lock_contract` requires **all** declared roles to sign — no per-pair / scoped-party
  contracts yet (three-role tasks, discussion #9).
- Contracts assume an **HTTP** interface (endpoints + `staging_url`); non-HTTP media
  (BLE/serial/mqtt/…) are not first-class yet (discussion #16).
- The GUI always runs the broker in **remote** mode, so `staging_url` requires `https` even
  for same-machine tasks (localhost not accepted in the contract).
- No image/screenshot sharing between agents yet.

## Upgrade / install
Standard checkout + `uv sync`; run `sys-buddy gui` (desktop) or `sys-buddy local` / `serve`.
No migrations required for a fresh install.
