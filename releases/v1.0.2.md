# sys-buddy v1.0.2 — 2026-07-23

Infrastructure release. No agent-visible or tool-contract changes.

## Added
- **Docker packaging.** A multi-stage root `Dockerfile` (built with `uv`) and
  `docker-compose.yml` run the broker as a single container, persisting the SQLite DB
  in a `sys-buddy-data` volume.
- **Devcontainer workspace** (`.devcontainer/`). A VS Code devcontainer that *is* the
  broker host: it mounts the repo and syncs the venv, and you run
  `uv run sys-buddy serve` (or `local`) inside it directly — there is no separate
  packaged-image container running alongside it. A `cloudflared` service exposes the
  workspace over a free quick tunnel (or a named tunnel via `CLOUDFLARE_TUNNEL_TOKEN`),
  and an optional `tailscale` sidecar joins it to a tailnet. All three share one
  compose network, so the tunnel/tailnet always reach whatever's listening on the
  devcontainer's `:8787` — no matter which process started it.

## Notes
- Packaging/dev-tooling only; the broker's MCP tools, wire contract, and dashboard
  behavior are unchanged from v1.0.1.
