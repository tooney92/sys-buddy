# sys-buddy v1.0.1 — 2026-07-23

Patch release. One fix, no new features, no schema changes.

## Fixed
- **Dashboard message thread was not ordered by creation time.** The thread merged
  chat messages with broker event-dividers (transitions, locks) and sorted them by
  **minute precision** only — so two items in the same minute, and any message↔event
  pair in the same minute, could appear out of order.
  - `api.py` now returns a raw `created_at` timestamp on every message (`_messages_for`)
    and every event (`_events_for`, as a 4th tuple element).
  - `ui.html` sorts the thread by that true timestamp (sub-second), with a stable
    tiebreak on the API's returned order (message id order).
  - The new sort is **backward-compatible**: if the API doesn't supply `ts` for every
    item (e.g. an old `api.py` still in memory before a broker restart, while `ui.html`
    is served fresh from disk), it falls back to the previous minute-based sort — so an
    in-flight dashboard never breaks.

## Notes
- Agent-facing message order was already correct (`check_messages` / `channel_history`
  order by message `id`); this only affected the read-only dashboard view.
- No DB migration. Takes full effect once the broker process is restarted (the running
  broker keeps serving the old behavior until then).
